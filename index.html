SLIIT Final Year Research Project 2024/25

Modular Multi-Agent Document AI: Adaptive Hierarchical Memory Networks for Intelligent Document Processing

A modular, multi-agent platform for intelligent document processing. The system combines Adaptive Hierarchical Memory Networks (AHMN), Explainable AI (XAI) with Human-in-the-Loop feedback, a six-layer Guardrails ensemble, and an Intelligent Document Processing Orchestrator to deliver accurate, transparent, and trustworthy information extraction from financial documents.

Supervisor Ms. Samadhi Samaratunga
Co-Supervisor Dr. Lakmini Abeywardhana
Group 4 Members
Domain NLP & Document AI

Research Domain

Domain

Exploring the intersection of Natural Language Processing, Explainable AI, and Intelligent Document Processing.

Literature Survey

Transformer-based architectures (Vaswani et al., 2017) have become the standard for NLP tasks including question answering and document comprehension. Models such as BERT (Devlin et al., 2019), BART (Lewis et al., 2020), and DeBERTa (He et al., 2021) achieve high accuracy but operate as opaque "black boxes." The field of Explainable AI (XAI) addresses this through techniques like LIME (Ribeiro et al., 2016) and SHAP (Lundberg & Lee, 2017). Multi-armed bandit algorithms, particularly Thompson Sampling (Thompson, 1933; Agrawal & Goyal, 2012), offer a principled framework for adaptive decision-making under uncertainty. Human-in-the-Loop (HITL) paradigms (Monarch, 2021) enable iterative system improvement through expert feedback.

In document processing, OCR systems often produce flattened text where table structures are lost (Xu et al., 2020). Financial documents present additional challenges with domain-specific formats, numeric precision requirements, and compliance needs. Existing systems lack integrated memory management, adaptive policy selection, and transparent explainability for generative question answering.

Research Gap

  • No existing IDP system combines adaptive memory with explainable generative Q&A and guardrailed processing in a modular architecture.
  • Current policy selection methods use static rules rather than learning from processing outcomes via bandit algorithms.
  • XAI techniques (LIME, SHAP) are rarely adapted for encoder-decoder generative models — most target classification tasks.
  • HITL feedback systems capture binary correctness but lack multi-dimensional assessment of explanation quality.
  • Memory management in IDP lacks drift-aware forgetting mechanisms that adapt to changing document distributions.

Research Problem

How can an intelligent document processing system adaptively select optimal processing policies based on document characteristics, provide transparent and verifiable explanations for its outputs, and continuously improve through structured human feedback — all within a modular, production-ready architecture?

Research Objectives

  1. Design a three-tier Adaptive Hierarchical Memory Network (STM, MTM, LTM) with Thompson Sampling for policy selection and drift-aware adaptive forgetting.
  2. Implement a generative XAI engine using BART encoder-decoder with cross-attention extraction and LIME for dual explainability.
  3. Build a six-layer Guardrails ensemble for schema validation, relevance checking, consistency verification, domain compliance, PII detection, and faithfulness scoring.
  4. Develop an Intelligent Document Processing Orchestrator (IDPO) that coordinates all agents through a configurable pipeline.
  5. Create a Human-in-the-Loop feedback subsystem with multi-dimensional quality assessment that feeds back into the memory system.
  6. Deploy the complete system as a containerised microservice architecture with a web-based frontend.

Methodology

The system follows a modular multi-agent architecture with eight Docker services communicating through REST APIs. Each agent operates independently and can be developed, tested, and scaled in isolation.

  • AHMN: Three-tier memory (Redis STM → PostgreSQL MTM → pgvector LTM) with Thompson Sampling, RABI prior transfer, and DAAF forgetting.
  • XAI-HITL: BART encoder-decoder for generative Q&A, cross-attention + LIME for explainability, Waterfall Pipeline for question routing.
  • Guardrails: SentenceTransformer (all-MiniLM-L6-v2) for semantic similarity, six-layer ensemble with config-driven policies.
  • IDPO: Pipeline orchestrator coordinating document flow across all agents.
  • Gateway: FastAPI reverse proxy with authentication and rate limiting.
  • Frontend: Next.js web application for document upload, Q&A, and feedback collection.
